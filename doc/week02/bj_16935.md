# 백준 16935번 배열 돌리기 3

## 문제
크기가 N×M인 배열이 있을 때, 배열에 연산을 R번 적용하려고 한다. 연산은 총 6가지가 있다. <br>
1번 연산은 배열을 상하 반전시키는 연산이다. <br>
2번 연산은 배열을 좌우 반전시키는 연산이다. <br>
3번 연산은 오른쪽으로 90도 회전시키는 연산이다. <br>
4번 연산은 왼쪽으로 90도 회전시키는 연산이다. <br>
5, 6번 연산을 수행하려면 배열을 크기가 N/2×M/2인 4개의 부분 배열로 나눠야 한다. 아래 그림은 크기가 6×8인 배열을 4개의 그룹으로 나눈 것이고, 1부터 4까지의 수로 나타냈다. <br>
5번 연산은 1번 그룹의 부분 배열을 2번 그룹 위치로, 2번을 3번으로, 3번을 4번으로, 4번을 1번으로 이동시키는 연산이다. <br>
6번 연산은 1번 그룹의 부분 배열을 4번 그룹 위치로, 4번을 3번으로, 3번을 2번으로, 2번을 1번으로 이동시키는 연산이다. 

## 입력
첫째 줄에 배열의 크기 N, M과 수행해야 하는 연산의 수 R이 주어진다. <br>
둘째 줄부터 N개의 줄에 배열 A의 원소 Aij가 주어진다. <br>
마지막 줄에는 수행해야 하는 연산이 주어진다. 연산은 공백으로 구분되어져 있고, 문제에서 설명한 연산 번호이며, 순서대로 적용시켜야 한다. <br>

## 출력
입력으로 주어진 배열에 R개의 연산을 순서대로 수행한 결과를 출력한다.

## 제한
- 2 ≤ N, M ≤ 100
- 1 ≤ R ≤ 1,000
- N, M은 짝수
- 1 ≤ Aij ≤ 108

## 정답 코드 (java)
```
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
	
	static int N, M, R;
	static int[][] arr;

	public static void main(String[] args) throws IOException {
		BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
		StringTokenizer st = new StringTokenizer(br.readLine(), " ");
		
		N = Integer.parseInt(st.nextToken());
		M = Integer.parseInt(st.nextToken());
		R = Integer.parseInt(st.nextToken());
		arr = new int[N][M];
		
		for (int n = 0; n < N; n++) {
			st = new StringTokenizer(br.readLine(), " ");
			for (int m = 0; m < M; m++) {
				arr[n][m] = Integer.parseInt(st.nextToken());
			}
		} // for
		
		st = new StringTokenizer(br.readLine(), " ");
		for (int i = 0; i < R; i++) {
			switch (Integer.parseInt(st.nextToken())) {
			case 1: upDown(); break;
			case 2: leftRight(); break;
			case 3: right90(); break;
			case 4: left90(); break;
			case 5: clock(); break;
			case 6: unClock(); break;
			} // switch
		} // for

		br.close();
		
		StringBuilder sb = new StringBuilder();
		for (int[] outArr : arr) {
			for (int intVar : outArr) {
				sb.append(intVar).append(" ");
			}
			sb.append("\n");
		} // for - result array
		System.out.println(sb.toString());
		
	} // main

	private static void upDown() {
		int tmp;
		for (int n = 0; n < arr.length/2; n++) {
			for (int m = 0; m < arr[0].length; m++) {
				tmp = arr[n][m];
				arr[n][m] = arr[N-1-n][m];
				arr[N-1-n][m] = tmp;
			}
		} // for
	} // upDown

	private static void leftRight() {
		int tmp;
		for (int n = 0; n < arr.length; n++) {
			for (int m = 0; m < arr[0].length/2; m++) {
				tmp = arr[n][m];
				arr[n][m] = arr[n][M-1-m];
				arr[n][M-1-m] = tmp;
			}
		} // for

	} // leftRight

	private static void right90() {
		int[][] arrCopy = new int[M][N];
		for (int n = 0; n < N; n++) {
			for (int m = 0; m < M; m++) {
				arrCopy[m][N-1-n] = arr[n][m];
			} // for - m
		} // for - n
		int tmp = M; M = N; N = tmp;
		arr = arrCopy;
	} // right90

	private static void left90() {
		int[][] arrCopy = new int[M][N];
		for (int n = 0; n < N; n++) {
			for (int m = 0; m < M; m++) {
				arrCopy[M-1-m][n] = arr[n][m];
			} // for - m
		} // for - n
		int tmp = M; M = N; N = tmp;
		arr = arrCopy;
	} // left90

	private static void clock() {
		int tmp;
		for (int n = 0; n < N/2; n++) {
			for (int m = 0; m < M/2; m++) {
				tmp = arr[n][m];
				arr[n][m] = arr[N/2+n][m];
				arr[N/2+n][m] = arr[N/2+n][M/2+m];
				arr[N/2+n][M/2+m] = arr[n][M/2+m];
				arr[n][M/2+m] = tmp;
			}
		}
	} // clock

	private static void unClock() {
		int tmp;
		for (int n = 0; n < N/2; n++) {
			for (int m = 0; m < M/2; m++) {
				tmp = arr[n][m];
				arr[n][m] = arr[n][M/2+m];
				arr[n][M/2+m] = arr[N/2+n][M/2+m];
				arr[N/2+n][M/2+m] = arr[N/2+n][m];
				arr[N/2+n][m] = tmp;
			}
		}
	} // unClock
}
```

## 회고
- 처음에는 각 연산에 대하여 메소드를 새로 생성할 생각을 못 했다. 메소드로 기능을 분리하니 확실히 코드 짜기가 더 수월했다.
- 3, 4번 연산을 수행할 때 배열의 크기가 바뀌는 것을 주의해야 했다.
- 5, 6번 연산을 수행할 때 한번에 요소 4개의 이동을 함께 하는 것이 코드도 깔끔하고 이해하기 쉬웠다.
