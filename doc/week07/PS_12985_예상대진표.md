# 프로그래머스 예상 대진표

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12985)

## 초기 접근
- 선수들을 넣을 큐를 만듦
- 1~N까지 큐에 넣음 (선수들을 넣음)
- 앞에서부터 두 명씩 꺼내고, 한 명을 넣음
- a 혹은 b를 만난 경우 무조건 a, b를 넣음
- 꺼낸 두 명이 a와 b면 break
- 센 라운드 수를 리턴

### 초기 코드
```
import java.util.ArrayDeque;
import java.util.Queue;

class Solution
{
    public int solution(int n, int a, int b)
    {
        int answer = 1;

        Queue<Integer> que = new ArrayDeque<>();
        
        for (int i = 1; i <= n; i++) {
        	que.offer(i);
        } //for - insert que
        
        while (que.size() > 1) {
        	int curA = que.poll();
        	int curB = que.poll();
        	
        	if (curA == a && curB == b || curA == b && curB == a) {
        		break;
        	} //if - meet a and b
        	
        	if (curA == a || curA == b) {
        		que.offer(curA);
        	} else if (curB == a || curB == b) {
        		que.offer(curB);
        	} else que.offer(0);
        	
        	if (que.size() == n / (answer*2)) answer++;
        } //while - round
        
        return answer;
    }
}
```

### 문제점
-  다음 라운드의 번호를 새로 만들지 않아서 번호 체계가 깨짐 (특히 0을 넣는 순간 실제 참가자 번호 구조가 완전히 깨짐)
-  정확한 라운드 판별에 어려움이 있음

## 개선
- 큐를 사용하지 않고도 간단하게 풀 수 있음
- 다음 라운드 번호: `(번호 + 1) / 2`
- a와 b의 라운드 번호를 계속 갱신하다가 둘이 같아지는 순간이 만나는 라운드가 됨
- 다른 참가자는 생각할 필요 없음

### 개선 코드
```
public class Solution {
	static int solution(int n, int a, int b)
    {
        int answer = 0;

        while (a != b) {
        	a = (a + 1) / 2;
        	b = (b + 1) / 2;
        	answer++;
        }
        
        return answer;
    }
}
```
