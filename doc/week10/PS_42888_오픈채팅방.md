# 프로그래머스 오픈채팅방

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42888)

## 접근
- 닉네임을 저장할 배열 -> 유저 id를 key로, 닉네임을 value로 갖는 `HashMap<String, String>` 생성
- 채팅창 출입 기록 저장 배열 -> 유저 id와 출입 기록을 저장할 `List<String[]>` 생성
  - 들어온 순서대로 저장하여 나중에 id로 해당 유저의 닉네임 찾아 같이 출력
- `Change` 인 경우 해당 유저의 id를 찾아 닉네임 변경 -> `replace()` 메소드 사용
- 정답 배열에 하나씩 저장 (정답 배열의 크기는 출입 기록 저장 배열의 크기와 같음)

### 정답 코드
```
import java.util.ArrayList;
import java.util.HashMap;
import java.util.List;
import java.util.StringTokenizer;

class Solution {
    public String[] solution(String[] record) {
        String[] answer = {};

        HashMap<String, String> names = new HashMap<>();
        List<String[]> inout = new ArrayList<>();

        for (String str : record) {
            StringTokenizer st = new StringTokenizer(str, " ");
            String order = st.nextToken();
            String id = st.nextToken();
            String name;

            switch (order) {
                case "Enter":
                    name = st.nextToken();
                    inout.add(new String[]{id, "님이 들어왔습니다."});
                    names.put(id, name);
                    break;
                case "Leave":
                    inout.add(new String[]{id, "님이 나갔습니다."});
                    break;
                case "Change":
                    name = st.nextToken();
                    names.replace(id, name);
                    break;
            } //switch - order
        } //for

        answer = new String[inout.size()];
        for (int i = 0; i < inout.size(); i++) {
            String id = inout.get(i)[0];
            String log = inout.get(i)[1];
            answer[i] = names.get(id) + log;
        } //for - input answer

        return answer;
    } //solution
}
```

### 회고
- 처음에는 출입 기록 저장 배열도 맵으로 생성하려고 했으나 출입 기록에는 같은 아이디인 사람이 들어와도 새로 저장해야 하기 때문에 리스트 자료구조로 변경하였다.
- `Leave`의 경우 주어진 문자열에 유저 닉네임이 없기 때문에 유저 닉네임 저장은 `Enter` 와 `Change` 인 경우에만 해주었다.
