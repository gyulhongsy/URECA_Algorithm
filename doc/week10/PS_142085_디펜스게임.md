# 프로그래머스 디펜스 게임

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/142085)

## 초기 접근
- 새 배열(sorted)에 enemy 배열을 복사한 뒤 내림차순 정렬 -> 무적권을 사용할 최소 적의 인원수를 K에 저장 (k-1번째)
- 배열을 돌면서 `n - enemy[i]`를 하는데 `k > 0 && enemy[i] >= K` 이면 `n - 0`, `k--`
  - 그렇지 않으면 `n - enemy[i]`
  - `n < 0` 이면 반복문을 빠져나오고 정답 리턴
  - 그렇지 않으면 `answer++`
 
### 초기 코드
```
import java.util.Arrays;
import java.util.Collections;

class Solution {
    public int solution(int n, int k, int[] enemy) {
        int answer = 0;
        int size = enemy.length;

        Integer[] sorted = new Integer[size];
        for (int i = 0; i < size; i++) sorted[i] = enemy[i];
        Arrays.sort(sorted, Collections.reverseOrder());

        int K = sorted[k-1];
        for (int i = 0; i < size; i++) {
            if (k > 0 && enemy[i] >= K) {
                n -= 0;
                k--;
            } else {
                n -= enemy[i];
            } //if ~ else

            if (n >= 0) answer++;
            else break;
        } //for

        return answer;
    }
}
```

### 문제점
- 기준점 K보다 적의 인원수가 큰 경우가 뒤쪽에 있는 경우 초반에 병사를 다 소진하여 무적권이 남게 됨

## 개선
- 우선순위 큐를 사용하여 과거의 지나온 적 수를 저장
- 병사 수가 마이너스가 되면 우선순위 큐에서 가장 적이 많았던 라운드를 꺼냄 (무적권 사용)
- 무적권 횟수가 모두 소모되고, 병사 수도 마이너스가 될 때까지 반복

### 개선 코드
```
import java.util.Collections;
import java.util.PriorityQueue;

class Solution {
    public int solution(int n, int k, int[] enemy) {
        int answer = 0;
        int size = enemy.length;

        PriorityQueue<Integer> queue = new PriorityQueue<>(Collections.reverseOrder());

        for (int i = 0; i < size; i++) {
            n -= enemy[i];
            queue.add(enemy[i]);
            answer++;

            if (n < 0) {
                if (k > 0) {
                    n += queue.poll();
                    k--;
                } else {
                    answer--;
                    break;
                }
            } //if - 병사 수 마이너스
        } //for

        return answer;
    }
}
```

### 회고
- 우선순위 큐는 다른 조건을 넣어주지 않으면 기본적으로 오름차순 정렬을 한다. -> `Collections.reverseOrder()` 메소드를 사용하여 내림차순 정렬로 변경해 주었다.
- 너무 단순 구현으로 해결하려고 했던 것 같다.
