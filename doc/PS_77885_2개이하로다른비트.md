# 프로그래머스 2개 이하로 다른 비트

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/77885)

## 접근 (1)
- 주어진 수를 이진수 문자열로 변환
- 주어진 수 + 1부터 1씩 늘려가며 이진수로 변환
- 주어진 수와 변환한 수를 비교하며 다른 비트의 수를 계산
- 다른 비트의 수가 2보다 작거나 같으면 바로 리턴

### 문제점
- 배열 안에 숫자들이 최대 10만개까지 들어있을 수 있고, 숫자는 최대 10<sup>15</sup>까지 주어지므로 대량의 데이터 처리 시 시간초과가 발생한다.
- 메모리에 썼다 버리는 임시 문자열 쓰레기들이 계속 쌓여 실행 시간이 급격히 느려짐
- 주어진 수와 비교할 수의 자릿수가 다름 -> format 함수를 통해 0을 채워 맞출 수 있으나 로직이 복잡해짐

### 접근 (2)
- 비트 연산자 `^`와 `bitCount()` 내장 함수로 간단하게 구현 가능
  - 비트 연산자 (^): 두 숫자 혹은 2진수의 각 비트를 비교해서 서로 다르면 1, 같으면 0을 반환
  - bitCount() 함수: 어떤 숫자를 2진수로 바꿨을 때 1이 몇 개 들어있는지 세어주는 자바 내장 함수 (Long, Integer 등)

### 초기 코드
```
class Solution {
    public long[] solution(long[] numbers) {
        long[] answer = new long[numbers.length];

        for (int i = 0; i < numbers.length; i++) {
            long num = numbers[i];
            for (long j = num+1; ; j++) {
                int diffCnt = Long.bitCount(num ^ j);

                if (diffCnt <= 2) {
                    answer[i] = j;
                    break;
                }
            }
        }

        return answer;
    }
}
```

### 문제점
- for문을 사용하여 1씩 더해주는 로직으로 루프를 돌면, 입력된 숫자 num이 약 1125조(2진수로 1이 50개 연속으로 붙어있는 수)일
  때 이 숫자보다 크면서 비트가 2개 이하로 다른 다음 숫자를 찾으려면 약 562조 번의 루프를 돌아야함. -> 약 65일이 걸림..

## 개선
- 루프를 사용하지 않고 2진수의 패턴을 파악하여 정답을 도출
- 짝수일 때
  - 짝수는 2진수로 바꾸었을 때 무조건 맨 끝자리가 0으로 끝남 -> 1을 더하면 1로 바뀜
  - 따라서 무조건 num + 1이 정답
- 홀수일 때
  - 홀수는 맨 끝자리가 1로 끝남
  - 비트를 가장 적게 바꾸는 방법 => 가장 뒤쪽에 있는 `01`을 `10`으로 변환
- 주어진 숫자를 2진수 문자열로 바꾸어 짝수인지 홀수인지 판단하여 진행

### 개선 코드
```
class Solution {
    public long[] solution(long[] numbers) {
        long[] answer = new long[numbers.length];

        for (int i = 0; i < numbers.length; i++) {
            StringBuilder sb = new StringBuilder("0" + Long.toBinaryString(numbers[i]));
            if (sb.charAt(sb.length()-1) == '0') {
                answer[i] = numbers[i] + 1;
            } else {
                for (int j = sb.length() - 1; j >= 0; j--) {
                    if (sb.charAt(j) == '0') {
                        sb.setCharAt(j, '1');
                        sb.setCharAt(j+1, '0');
                        break;
                    }
                } //for

                Long ansNum = Long.parseLong(sb.toString(), 2);
                answer[i] = ansNum;
            } //if ~ else
        } //for - numbers

        return answer;
    }
}
```

### 회고
- 문자열을 숫자로 바꾸는 `parseLong/Int()` 메소드를 사용할 때 파라미터를 하나 더 넣어 몇진법 숫자로 변환할 건지 선택할 수 있다는 것을 알게되었다.
- 숫자를 이진수 문자열로 바꿀 때 1로만 이루어진 경우를 생각해 제일 앞에 0을 추가해 주었다.
- 숫자가 매우 클 때에는 1씩 더하는 탐색 과정을 과감히 버리고, 비트나 패턴을 먼저 그려봐야 한다는 것을 배웠다.
