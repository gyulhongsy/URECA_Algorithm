# 프로그래머스 2 x n 타일링

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12900)

## 접근
- 동적 프로그래밍으로 n=1인 경우부터 n까지 구하였다.
  - n == 1 : 세로 타일 한 가지
  - n == 2 : 세로 타일 두개를 이어 붙이거나 가로 타일 두개를 이어 붙인 두 가지
  - n == 3 : n == 2의 경우에 세로 타일 한 개를 붙임 + n == 1의 경우에 가로 타일 두 개를 붙임 => 총 3가지
  - n == 4 : n == 3의 경우에 세로 타일 한 개를 붙임 + n == 2의 경우에 가로 타일 두 개를 붙임 => 총 5가지
  - n == 5 : n == 4의 경우에 세로 타일 한 개를 붙임 + n == 3의 경우에 가로 타일 두 개를 붙임 => 총 8가지
  - 따라서 점화식은 다음과 같다.
    ```
    dp[i] = dp[i-2] + dp[i-1];
    ```

### 초기 코드
```
class Solution {
    public int solution(int n) {
        int DIV = 1_000_000_007;

        int[] dp = new int[n+1]; //0번지 버림

        dp[1] = 1;
        dp[2] = 2;

        for (int i = 3; i <= n; i++) {
            dp[i] = dp[i-1] + dp[i-2];
        }

        return dp[n] % DIV;
    }
} 
```

### 문제점
- 문제에서 주어진 수로 나누느 것을 마지막  `return`할 때 딱 한 번만 함 -> n이 46쯤만 되어도 값이 기하급수적으로 커져 오버플로우가 발생함
- `n = 1` 일 때의 IndexOutOfBoundsException -> n = 1 이 들어오면 dp[2]를 호출할 때 인덱스를 벗어나게 되어 에러 발생

## 개선
- 더할 때마다 매번 나눠서, 그 나머지 값들을 dp 배열에 저장
- n == 1 일 때는 바로 1을 리턴하여 종료

### 개선 코드
```
class Solution {
    public int solution(int n) {
        if (n == 1) return 1;
        
        int DIV = 1_000_000_007;
        int[] dp = new int[n+1]; //0번지 버림

        dp[1] = 1;
        dp[2] = 2;

        for (int i = 3; i <= n; i++) {
            dp[i] = (dp[i-1] + dp[i-2]) % DIV;
        }

        return dp[n];
    }
}
```

### 회고
- 이전에 산모양 타일링이라는 문제를 푼 경험이 있어서 접근이 어렵지는 않았다.
- 배열의 인덱스 범위를 벗어나지는 않는지 꼼꼼히 확인할 필요가 있었다.
- 어떤 수로 나눈 나머지를 리턴하라는 문제가 나오면 계산 중간에 값이 자료형의 범위를 초과할테니 매번 나누면저 진행해야겠다.
