# 프로그래머스 아이템 줍기

[문제 링크](https://gemini.google.com/app/c412dc7d90ef6422)

## 접근
- 겹쳐진 다각형의 '가장 바깥쪽 외곽선(테두리)'만 따라가며 출발지에서 목표점까지의 최단 거리를 구한다 -> bfs 이용
- 일반적인 bfs를 그대로 돌리면, ㄷ자 모양의 1칸 간격 길에서 로봇이 옆 길로 벽을 뚫고 가버린다 -> 좌표를 2배로 늘리기
  ```
  확대 전 : 1칸 간격 ㄷ자          2배 확대 후 : 2칸 간격
       ■    ■                         ■    □    ■   <-- 가운데에 '□(0)'이라는
       └───┘                          │         │       확실한 차단벽이 생김!
                                      └────┴────┘
  ```
  - 모든 좌표를 늘리면 1칸 간격이었던 길이 2칸 간격으로 벌어지고, 그 사이에 빈 공간(벽)이 생기면서 벽을 뚫는 일이 없어짐
  
#### 구현 로직 4단계
1. 2차원 배열 넉넉하게 선언
  - 좌표의 최댓값이 50이므로 2배 확대하면 100이 됨
  - 인덱스 에러를 방지하고 완충 지대를 두기 위해 테두리를 두름 -> `int[102][102]` 크기의 배열 선언
2. 다각형의 테두리 그리기
  - 지도의 상태를 0(땅), 1(테두리), 2(사각형 내부)로 정의하고 사각형 좌표를 2배 늘려 순환
  - 칠하려는 자리가 이미 남의 사각형 내부(2)라면 건드리지 말고 패스
  - 테두리가 아닌 안쪽 구간이라면 2로 덮어쓰기
  - 위 1, 2번이 아니라면 테두리로 판단(1)
3. 테두리만 골라서 bfs
  - 두 배 늘린 출발점에서 시작 (좌표도 두 배 늘렸으므로)
  - 큐에 좌표와 거리를 함께 집어넣는다
4. 구한 거리를 반으로 나눠 최종 거리를 구함 (지도를 2배 늘려 걸었으므로 이동 거리는 2배가 됨)

### 정답 코드
```
import java.util.ArrayDeque;
import java.util.Queue;

public class Solution {
    public int solution(int[][] rectangle, int characterX, int characterY, int itemX, int itemY) {
        int[][] map = new int[102][102];

        for (int[] r : rectangle) {
            int x1 = r[0] * 2, y1 = r[1] * 2;
            int x2 = r[2] * 2, y2 = r[3] * 2;

            for (int i = x1; i <= x2; i++) {
                for (int j = y1; j <= y2; j++) {
                    if (x1 < i && x2 > i && y1 < j && y2 > j) map[i][j] = 2;
                    else if (map[i][j] != 2) map[i][j] = 1;
                }
            }
        } //for - 사각형들을 도화지에 그리기 (테두리: 1, 내부: 2)

        int startX = characterX * 2, startY = characterY * 2;
        int targetX = itemX * 2, targetY = itemY * 2;

        Queue<int[]> queue = new ArrayDeque<>();
        boolean[][] visited = new boolean[102][102];
        int[] dx = {-1, 1, 0, 0};
        int[] dy = {0, 0, -1, 1};

        queue.add(new int[] {startX, startY, 0});
        visited[startX][startY] = true;

        while (!queue.isEmpty()) {
            int[] cur = queue.poll();
            int cx = cur[0], cy = cur[1], dist = cur[2];

            if (cx == targetX && cy == targetY) return dist / 2;

            for (int d = 0; d < 4; d++) {
                int nx = cx + dx[d];
                int ny = cy + dy[d];

                if (nx >= 0 && nx < 102 && ny >= 0 && ny < 102) {
                    if (map[nx][ny] == 1 && !visited[nx][ny]) {
                        visited[nx][ny] = true;
                        queue.add(new int[] {nx, ny, dist+1});
                    }
                }
            }
        } //while - bfs

        return -1;
    }
}
```

### 회고
- bfs 로직 구현 자체는 어렵지 않았지만 처음에 지도의 좌표를 2배 늘린다는 생각은 하지 못했다.
- `if-else`로 계속 예외처리를 하려고 했다면 상당히 복잡했을 것 같다.
- 데이터가 놓여 있는 공간의 스케일 자체를 바꿔버리는 역발상이 중요했다.
