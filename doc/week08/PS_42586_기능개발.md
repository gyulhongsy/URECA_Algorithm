# 프로그래머스 기능개발

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42586)

## 접근
- 앞에서부터 각 작업별로 배포까지 필요한 일수를 계산하여 큐에 넣음
- 큐에서 하나씩 꺼내어 뒤의 일수를 확인해 현재 일수보다 적거나 같은 날짜 모두 꺼냄
- 꺼낸 만큼 cnt를 더하고, 현재 일수보다 큰 날짜가 나오면 cnt를 결과 배열에 저장
- 큐가 빌 때까지 반복

### 정답 코드
```
import java.util.ArrayDeque;
import java.util.ArrayList;
import java.util.Queue;

class Solution {
    public int[] solution(int[] progresses, int[] speeds) {
        ArrayList<Integer> answer = new ArrayList<>();

        Queue<Integer> que = new ArrayDeque<>();

        for (int i = 0; i < progresses.length; i++) {
            int tmp = (100 - progresses[i]) % speeds[i];
            int days = 0;
            if (tmp == 0) days = (100 - progresses[i]) / speeds[i];
            else days = (100 - progresses[i]) / speeds[i] + 1;
            que.add(days);
        } //for - calc days

        while (!que.isEmpty()) {
            int curMax = que.poll();
            int cnt = 1;

            while (!que.isEmpty()) {
                if (que.peek() <= curMax) {
                    que.poll();
                    cnt++;
                }
                else break;
            } //while - 뒤의 작업 확인

            answer.add(cnt);
        } //while

        return answer.stream().mapToInt(i -> i).toArray();
    }
}
```

### 회고
- 처음에는 진행도를 일수별로 올려가며 확인하려고 했지만 너무 비효율적인 방법이라는 생각이 들어 로직을 변경하게 되었다.
- 현재 작업을 빼고 뒤의 작업을 확인하는 과정에서 반복문 탈출 조건을 설정해 주지 않아 무한 루프가 돌았었다.
- 정답 배열의 크기를 미리 알 수 없고, 원소를 하나씩 추가해야 해서 `ArrayList` 를 사용하였다.
