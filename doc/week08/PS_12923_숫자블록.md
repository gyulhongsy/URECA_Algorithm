# 프로그래머스 숫자 블록

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/12923)

## 초기 접근
- 이 문제는 주어진 구간에서 현재 인덱스 숫자의 약수 중 가장 큰 수를 구하면 되는 문제이다(자기 자신 제외).
- for문을 2부터 숫자/2 까지 돈다
- for문을 돌면서 제일 먼저 나누어 떨어지는 경우가 있으면 그 몫이 가장 큰 약수
- 약수를 발견하면 break, for문에서 한번도 나누어 떨어지지 않았다면 소수 => 배열에는 1이 들어가게 됨

### 초기 코드
```
import java.util.Arrays;

class Solution {
    public int[] solution(long begin, long end) {
        int[] answer = new int[(int)(end - begin + 1)];

        Arrays.fill(answer, 1);
        answer[0] = 0;

        for (int i = 1; i < answer.length; i++) {
            int num = i + 1;
            for (int j = 2; j <= num / 2; j++) {
                if ((num) % j == 0) {
                    answer[i] = (num) / j;
                    break;
                } //if - 제일 큰 약수를 찾으면
            } //for - j
        } //for - i

        return answer;
    }
}
```

### 문제점
- for문을 `num / 2` 까지가 아닌 `Math.sqrt()` 까지만 돌아도 됨
- 제일 큰 약수는 1000만보다 작아야 함 => 1000만 보다 큰 숫자면 더 작은 약수를 구해야 함
- 주어진 구간에서 시작 숫자에 상관 없이 1부터의 약수 값을 다 구하고 있음

## 개선
- for문의 범위를 수정
- 1000만보다 작은 약수를 구하는 조건 추가
- 1부터가 아닌 주어진 구간의 시작 숫자부터 약수를 찾도록 변경

### 개선 코드(1)
```
class Solution {
    public int[] solution(long begin, long end) {
        int[] answer = new int[(int)(end - begin + 1)];

        for (int i = 0; i < answer.length; i++) {
            long num = begin + i; //실제 숫자

            if (num == 1) {
                answer[i] = 0;
                continue;
            }

            answer[i] = 1;

            for (long j = 2; j <= Math.sqrt(num); j++) {
                if (num % j == 0) {
                    if (num / j <= 10000000) {
                        answer[i] = (int)(num / j);
                    } else {
                        answer[i] = (int)j;
                    } //if ~ else - 1000만보다 작은 가장 큰 약수
                    break;
                } //if - 약수 후보 발견
            } //for - j
        } //for - i

        return answer;
    }
}
```

### 문제점
- 테스트 케이스는 모두 통과했으나 효율성 테스트에서 0점을 받음
