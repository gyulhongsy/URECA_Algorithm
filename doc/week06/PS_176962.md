# 프로그래머스 과제 진행하기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/176962)

## 초기 접근
- 스택에 지금 시작한 과제를 넣음
- 현재 시간(입력받는 다음 데이터)을 기준으로 스택에 넣은 과제의 진행 시간과 비교하여
- 시간 안에 과제를 끝내면 스택에서 제거, 끝내지 못하면 진행 시간을 갱신한 뒤 새로운 과제 시작(스택에 넣음)
- 위 과정을 주어진 데이터 수 만큼 반복하고 스택에 과제가 남아있으면 하나씩 빼면서 종료시킴
- 이름과 진행 시간 필드를 가진 과제 클래스 사용

### 초기 코드
```
import java.time.Duration;
import java.time.LocalTime;
import java.util.Stack;

public class Solution {
	
	static class Work {
		String name;
		int time;
		
		public Work(String name, int time) {
			this.name = name;
			this.time = time;
		}
	}
	
	public String[] solution(String[][] plans) {
        String[] answer = new String[plans.length];
        Stack<Work> stack = new Stack<>();
        
        LocalTime prevTime, curTime;
        stack.push(new Work(plans[0][0], Integer.parseInt(plans[0][2])));
        prevTime = LocalTime.parse(plans[0][1]);
        int idx = 0;
        
        for (int i = 1; i < plans.length; i++) {
        	curTime = LocalTime.parse(plans[i][1]);
        	
        	int diff = (int)Duration.between(prevTime, curTime).toMinutes();
        	if (diff >= stack.peek().time) {
        		answer[idx++] = stack.pop().name;
        	} else {
        		stack.peek().time -= diff;
        	} //if ~ else - 과제 하는 시간
        	stack.push(new Work(plans[i][0], Integer.parseInt(plans[i][2])));
        	prevTime = curTime;
        } //for
        answer[idx] = stack.pop().name;
        
        return answer;
    }
}
```

### 문제점
- 새로 들어온 과제를 다 하고 남은 시간에 이전 과제 한 개를 끝내도 시간이 남는 경우 다른 과제를 하지 않고 진행됨
- 과제가 시간순으로 정렬되어 있지 않음. → 변수를 하나 더 만들어서 입력 받은 과제를 먼저 정렬한 뒤 사용해야 할 듯

## 개선 방향
- `ArrayList` 를 사용해 입력 받은 과제들을 저장, 시작 시간 순으로 정렬 → 후에 이 자료를 사용
- 시작 시간 순 정렬을 위해 과제 클래스에 시작 시간을 저장하는 필드 추가
- `while (!stack.isEmpty())` 를 사용해 남은 시간 동안 처리할 수 있는 모든 과제를 처리

### 개선 코드
```
import java.time.Duration;
import java.time.LocalTime;
import java.util.ArrayList;
import java.util.Stack;

class Solution {
    static class Work {
		String name;
		LocalTime start;
		int time;
		
		public Work(String name, LocalTime start, int time) {
			this.name = name;
			this.start = start;
			this.time = time;
		} //Constructor
	}
	
	public String[] solution(String[][] plans) {
        String[] answer = new String[plans.length];
        Stack<Work> stack = new Stack<>();
        
        ArrayList<Work> works = new ArrayList<>();
        for (String[] p : plans) {
        	works.add(new Work(p[0], LocalTime.parse(p[1]), Integer.parseInt(p[2])));
        }
        works.sort((o1, o2) -> o1.start.compareTo(o2.start));
        
        LocalTime prevTime, curTime;
        stack.push(works.get(0));
        prevTime = works.get(0).start;
        int idx = 0;
        
        for (int i = 1; i < plans.length; i++) {
        	curTime = works.get(i).start;
        	
        	int diff = (int)Duration.between(prevTime, curTime).toMinutes();
        	
        	while (!stack.isEmpty() && diff >= stack.peek().time) {
        		diff -= stack.peek().time;
        		answer[idx++] = stack.pop().name;
        	} //while - 사이 시간 과제 처리
        	
        	if (!stack.isEmpty()) {
        		stack.peek().time -= diff;
        	} //if - 남은 시간 보다 과제에 요구되는 시간이 더 큰 경우
        	
        	stack.push(works.get(i));
        	prevTime = curTime;
        } //for
        
        while(!stack.isEmpty()) {
        	answer[idx++] = stack.pop().name;
        } //while - 남아 있는 과제들 처리
        
        return answer;
    }
}
```

### 회고
- 최근에 멈춘 과제부터 다시 시작한다는 점에서 스택 자료구조를 사용해야 한다는 생각이 들었다.
- 처음에는 남는 시간 동안 이전에 멈춘 과제들을 다시 진행해야 한다는 점을 놓쳤다.
- 이후 diff 시간을 계속 감소시키며 스택의 과제를 반복 처리하는 과정으로 수정했다.
- 시간 순으로 주어진 데이터를 정렬할 때 `LocalTime` 은 따로 정수형으로 변환하는 과정을 거치지 않아도 `compareTo()` 메소드를 통해 정렬할 수 있음을 알게 되었다.
