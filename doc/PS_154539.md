# 프로그래머스 뒤에 있는 큰 수 찾기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/154539)

## 초기 접근
- 이중 for문을 사용하여 현재 수 기준으로 뒷 부분을 계속 확인하며 큰 수가 생기면 갱신한다. (브루트포스 방식)

### 초기 코드
```
import java.util.Arrays;

class Solution {
    public int[] solution(int[] numbers) {
        int[] answer = new int[numbers.length];
        Arrays.fill(answer, -1);
        
        for(int i = 0; i < answer.length; i++) {
        	for (int j = i; j < numbers.length - 1; j++) {
        		if (numbers[j+1] > numbers[i]) {
        			answer[i] = numbers[j+1];
        			break;
        		}
        	}
        }
        
        return answer;
    }
}
```

### 문제점
- 초기 코드는 이중 for문을 사용하여 시간복잡도는 O(n<sup>2</sup>)으로, numbers의 크기가 매우 큰 경우 시간이 매우 오래 걸린다.
- 전체 테스트 케이스 중 4개의 테스트 케이스에서 시간 초과가 발생하였다.

## 개선 방향
- 스택 자료구조를 사용하고 배열은 한 번만 돈다.
- 스택에는 배열의 인덱스 값을 넣고, 뒤에 있는 큰 수를 찾은 경우 해당 수를 answer의 인덱스 값에 저장한 뒤 스택에서 뺀다.
- 위 과정을 배열을 다 돌때까지 반복.
- 배열을 다 돌았는데도 스택에 데이터가 남아 있으면 → 뒤에 있는 큰 수를 찾지 못한 경우.
- `stack.pop()` 을 하면서 해당 인덱스 자리에 -1 넣음
- 시간 복잡도: O(n)

### 개선 코드
```
import java.util.Stack;

class Solution {
    public int[] solution(int[] numbers) {
        int[] answer = new int[numbers.length];
        
        Stack<Integer> stack = new Stack<>();
        
        stack.push(0);
        for (int i = 1; i < numbers.length; i++) {
        	while (!stack.isEmpty() && numbers[i] > numbers[stack.peek()]) {
        		answer[stack.pop()] = numbers[i];
        	}
        	stack.push(i);
        }
        
        if (!stack.isEmpty()) {
        	while (!stack.isEmpty()) answer[stack.pop()] = -1;
        }
        
        return answer;
    }
}
```

### 회고
- for 문과 while 문이 함께 있어서 이중 반복문으로 보일 수 있으나 결론적으로 각 원소를 스택에서 한 번씩만 넣고 빼기 때문에 전체 시간 복잡도가 현저히 줄어들게 되었다.
- 또한 현재 숫자가 여러 이전 원소의 정답이 될 수 있다는 점을 통해 이미 확인한 값을 재활용할 방식으로 개선할 수 있었다.
