# 프로그래머스 택배상자

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/131704)

## 초기 접근
- 주어진 order 배열을 돌며 컨테이너 벨트에 오는 상자와 비교
- 경우에 따라 보조 벨트에 올릴 것인지, 트럭에 실을 것인지 결정
- 보조 벨트는 `stack` 자료구조를 사용

### 초기 코드
```
import java.util.Stack;

public class Solution {
    static int solution(int[] order) {
        int answer = 0;

        Stack<Integer> stack = new Stack<>();

        for (int i = 0; i < order.length; i++) {
            if (order[i] == i+1) {
                answer++;
            } else if (!stack.isEmpty() && stack.peek() == order[i]) {
                stack.pop();
                answer++;
            } else {
                stack.push(i+1);
            }
        } //for

        return answer;
    }
}
```

### 문제점
- for문 기준 i 하나로 order와 메인 벨트의 상자 번호를 동시에 제어하려고 하여 시점이 맞지 않음

## 개선
- for문을 order 배열이 아니라 메인 벨트에서 1번부터 차례로 들어오는 상자를 기준으로 돌림
- 택배 기사님이 기다리는 순서는 별도의 변수(`orderIdx`)로 관리
- 일단 보조 벨트(스택)에 밀어 넣고, 트럭에 실을 수 있을 때까지 계속 실음

### 개선 코드
```
import java.util.Stack;

class Solution {
    static int solution(int[] order) {
        int answer = 0;
        Stack<Integer> stack = new Stack<>();
        int orderIdx = 0;

        for (int i = 1; i <= order.length; i++) {
            stack.push(i);

            while(!stack.isEmpty() && stack.peek() == order[orderIdx]) {
                stack.pop();
                answer++;
                orderIdx++;
            } //while - 트럭에 실을 수 있는 만큼 싣기
        } //for - 메인 벨트에서 박스가 차례로 들어옴

        return answer;
    }
}
```

### 회고
- 간단한 문제였지만 접근을 다르게 해서 많이 돌아간 느낌이었다. 코드를 짜다가 조건 처리가 너무 꼬인다면, 내가 설정한 반복문의 시점이나 기준 자체가 잘못되지 않았는지 의심해보는 습관을 가져야겠다.
- 반복문을 순회할 때 if-else로 복잡하게 작성하기 보다 일단 스택에 밀어 넣고, while문을 사용하여 조건이 맞을때까지 연속으로 빼는 구조가 훨씬 단순하고 명확하게 만든다는 것을 알게 되었다.
