# 프로그래머스 스킬트리

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/49993)

## 초기 접근
- `indexOf()` 메소드를 활용하여 각 스킬들이 스킬트리의 몇 번째에 있는지 찾는다
- 찾은 인덱스를 비교하여 순차적인지 확인하여 정답을 구한다

### 초기 코드
```
import java.util.ArrayList;
import java.util.List;

class Solution {
    public int solution(String skill, String[] skill_trees) {
        int answer = 0;

        List<Character> chars = new ArrayList<>();
        for (char c : skill.toCharArray()) {
            chars.add(c);
        } //for - insert chars

        List<Integer> idx = new ArrayList<>();
        int prevIdx = 0;
        for (String str : skill_trees) {
            for (int i = 0; i < chars.size(); i++) {
                if (i == 0) {
                    prevIdx = str.indexOf(chars.get(i));
                    idx.add(prevIdx);
                }
                else {
                    idx.add(str.indexOf(chars.get(i), prevIdx + 1));
                    prevIdx = str.indexOf(chars.get(i));
                }
            } //for - 스킬트리에 해당 스킬이 몇 번째에 있는지 구하기

            boolean isMatched = true;
            for (int i : idx) {
                if (i == -1) {
                    isMatched = false;
                    break;
                } //if - 순서가 다름을 발견
            } //for - 순서대로 포함되어 있는지

            if (isMatched) answer++;
        }

        return answer;
    }
}
```

### 문제점
- `List<Integer> idx` 와 `int prevIdx` 가 스킬트리를 순회하는 for문 바깥에 선언되어 있어 첫 번째 스킬트리를 검사하고 난 뒤 idx 리스트가 비워지지 않고 계속 추가되어서 완전히 엉뚱한 위치부터 검색을 시작하게 됨
- 정상 스킬트리지만 스킬을 도중에 멈춘 경우도 가짜로 판정해버려 오답이 나옴
- `prevIdx` 갱신 시점에 오류가 있어 논리적으로 어긋나게 됨

## 개선
- 스킬트리를 하나씩 꺼내볼 때마다 변수들이 새롭게 초기화되도록 `List<Integer> idx` 와 `int prevIdx` 를 for문 안쪽으로 위치 변경
- -1이 나왔다고 무조건 틀린 것이 아닌 순서가 틀렸을 때만 오답으로 처리
- `curIdx` 변수를 추가해 `prevIdx` 갱신 시점 오류를 개선

### 개선 코드
```
class Solution {
    public int solution(String skill, String[] skill_trees) {
        int answer = 0;

        for (String str : skill_trees) {
            boolean isMatched = true;
            int prevIdx = -1; //앞선 스킬의 위치
            boolean isMissed = false; //선행 스킬을 건너뛰었는지 체크

            for (char c : skill.toCharArray()) {
                int curIdx = str.indexOf(c);

                if (curIdx == -1) { //현재 스킬을 안배움
                    isMissed = true;
                } else { //현재 스킬을 배움
                    if (isMissed || curIdx < prevIdx) {
                        isMatched = false;
                        break;
                    } //if - 선행 스킬을 안배웠거나 순서가 바뀜

                    prevIdx = curIdx; //무사히 통과 -> 다음 스킬과 비교하기 위해 현재 위치 저장
                } //if
            } //for - 스킬 한 글자씩 검사

            if (isMatched) answer++;
        } //for - 스킬 트리에서 하나씩 검사

        return answer;
    }
}
```

### 회고
- 반복문 안에서 쓰이는 결과 누적 리스트나 변수를 검사할 때마다 반드시 초기화되어야 한다는 점을 느꼈다.
- 정답 처리를 할 때 조건을 좀 더 까다롭게 검사해야겠다.

## 개선 - 정규식
- 기존의 코드를 개선할 방법을 계속 찾다가 더 직관적으로 코드를 짤 수 있는 **정규식** 방법으로 다시 짜보았다.

### 개선 코드 - 정규식
```
class Solution {
    public int solution(String skill, String[] skill_trees) {
        int answer = 0;

        //남길 문자들(스킬)을 제외한 나머지 모든 문자를 찾는 정규식 생성
        String regex = "[^" + skill + "]";

        for (String st : skill_trees) {

            //스킬트리에서 skill과 상관없는 잡다한 스킬들을 빈 문자열로 치환하여 모두 제거
            String filtered = st.replaceAll(regex, "");

            //남은 스킬들이 정해진 순서인지 확인
            if (skill.startsWith(filtered)) answer++;
        }

        return answer;
    }
}
```

### 회고
- 정규식과 문자열 메소드들에 대해 알고있으면 활용할 수 있는 문제들이 꽤 있을 것 같다.
- 정규표현식과 `startsWith` 을 조합한 방식처럼 역발상으로 접근하는 것도 아주 강력한 문제 해결 방식임을 알게 되었다.
