# 프로그래머스 가장 큰 수

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/42746)

## 초기 접근
- 순열 알고리즘을 사용해 주어진 숫자 배열에서 숫자 배열의 크기만큼 수를 뽑음
- 뽑은 수를 문자열로 변환하여 이어붙임
- 이어 붙인 수를 다시 숫자로 변환하여 최대값인지 비교하여 리턴

### 초기 코드
```
public class Solution {

    static int MAX = Integer.MIN_VALUE;
    static int n;

    static int[] input;
    static int[] picked;
    static boolean[] selected;

    public String solution(int[] numbers) {
        String answer = "";

        n = numbers.length;
        input = new int[n];
        picked = new int[n];
        selected = new boolean[n];

        for (int i = 0; i < n; i++) input[i] = numbers[i];

        permutation(0);

        answer = String.valueOf(MAX);
        return answer;
    } //solution

    private void permutation(int idx) {
        if (idx >= n) {
            StringBuilder sb = new StringBuilder();
            for (int i = 0; i < n; i++) {
                sb.append(String.valueOf(picked[i]));
            } //for - 숫자 연결
            int num = Integer.parseInt(sb.toString());
            MAX = Math.max(num, MAX);
        } //if - base case

        for (int i = 0; i < n; i++) {
            if (selected[i]) continue;

            picked[idx] = input[i];
            selected[i] = true;
            permutation(idx+1);
            selected[i] = false;
        } //for - pick number
    } //permutation
}
```

### 문제점
- 주어지는 배열의 길이가 10만개라서 만약 모든 숫자를 이어 붙이면 10만 자릿수가 됨. 하지만 Integer형으로 변환하려고 해서 에러가 나고, 시간 초가가 됨

## 개선
- 기존에 주어진 숫자 배열을 미리 정렬하여 가장 큰 수를 찾는다
  - `Comparator` 와 `CompareTo` 메소드를 사용하여 정렬 알고리즘 작성

### 개선 코드
```
import java.util.Arrays;

public class Solution {
    public String solution(int[] numbers) {
        int n = numbers.length;

        String[] strNumbers = new String[n];
        for (int i = 0; i < n; i++) {
            strNumbers[i] = String.valueOf(numbers[i]);
        }

        Arrays.sort(strNumbers, (o1, o2) -> (o2+o1).compareTo(o1+o2));

        if (strNumbers[0].equals("0")) return "0"; //입력 받은 숫자에 0만 있는 경우

        StringBuilder sb = new StringBuilder();
        for (String str : strNumbers) {
            sb.append(str);
        }

        return sb.toString();
    }
}
```

### 회고
- 입력 값의 범위가 매우 클 때는 완전 탐색(순열/조합)으로 풀 수 없다는 것을 배웠다.
- 이번 문제 같은 경우에는 커스텀 정렬을 바로 생각했다면 더 쉽게 풀 수 있었을 것이다.
