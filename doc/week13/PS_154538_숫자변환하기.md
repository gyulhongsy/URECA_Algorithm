# 프로그래머스 숫자 변환하기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/154538)

## 접근
- bfs를 이용하여 세 가지 연산을 한 숫자를 모두 큐에 넣는다
- 큐에서 하나씩 꺼내며 y와 비교하여 다음 연산 실행
  - 결과 < y -> 큐에 결과와 횟수+1 넣기
  - 결과 == y -> 횟수 리턴
  - 결과 > y and que.isEmpty() -> return -1
 
### 초기 코드
```
import java.util.ArrayDeque;
import java.util.Queue;

class Solution {
    public int solution(int x, int y, int n) {
        Queue<int[]> queue = new ArrayDeque<>();
        queue.add(new int[] {x, 0});

        while (!queue.isEmpty()) {
            int[] tmp = queue.poll();
            int num = tmp[0];
            int cnt = tmp[1];

            if (num < y) {
                queue.add(new int[] {num+n, cnt+1});
                queue.add(new int[] {num*2, cnt+1});
                queue.add(new int[] {num*3, cnt+1});
            } else if (num == y) {
                return cnt;
            } else if (num > y && queue.isEmpty()) {
                return -1;
            } //if ~ else - 숫자와 y 비교
        } //while - bfs

        return -1;
    }
}
```

### 문제점
- 몇 개의 테스트케이스서 메모리 초과가 발생함
  - 원인: 이미 큐에 넣었던 숫자를 또 넣음 (방문 처리가 되지 않음) -> 숫자들이 기하급수적으로 많아지고 결국 메모리 초과 발생
- 처음 숫자와 도달 숫자가 같은 경우를 체크하지 않음 -> 큐에 넣은 뒤에는 연산을 한 뒤 결과 숫자와 비교하기 때문에 결국 -1이 반환됨

## 개선
- `visited[]` 배열을 사용하여 똑같은 수를 또 넣지 않도록 방지
- 메소드 첫 부분에 처음 숫자와 도달 숫자가 같은 경우를 처리하는 코드 추가

### 개선 코드
```
import java.util.ArrayDeque;
import java.util.Queue;

class Solution {
    public int solution(int x, int y, int n) {
        if (x == y) return 0;

        Queue<int[]> queue = new ArrayDeque<>();
        boolean[] visited = new boolean[1_000_001];

        queue.add(new int[] {x, 0});
        visited[x] = true;

        while (!queue.isEmpty()) {
            int[] tmp = queue.poll();
            int num = tmp[0];
            int cnt = tmp[1];

            int[] nextNums = {num+n, num*2, num*3};

            for (int next : nextNums) {
                if (next == y) return cnt+1;

                if (next < y && !visited[next]) {
                    visited[next] = true;
                    queue.add(new int[] {next, cnt+1});
                } //if - insert next
            } //for - next
        } //while - bfs

        return -1;
    }
}
```

### 회고
- bfs 알고리즘을 사용하면서 방문 처리를 하지 않은 점이 아쉬웠다.
- 시작 숫자와 최종 숫자가 같은 경우처럼 코드를 작성할 때 놓친 예외 처리가 없는지 꼼꼼히 확인해야겠다.
