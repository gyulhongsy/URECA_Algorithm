# 프로그래머스 입국심사

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/43238)

## 접근
- 시뮬레이션으로 사람을 한명씩 대기열에 넣어서 차례차례 시간을 계산한다. => 대기하는 사람 수가 최대일 때 10억명이기 때문에 무조건 시간 초과가 발생함
- 이분 탐색을 사용하여 정답을 찾아가야 함
- 정답이 될 수 있는 시간의 범위를 양 끝단으로 잡아놓고, 반씩 쪼개가며 최적의 시간 탐색
- 최소 시간 `left` : 1분 (논리적인 최솟값)
- 최대 시간 `right` : 가장 심사가 느린 심사관이 n명을 혼자서 다 심사할 때 걸리는 시간 (최악의 경우)
- 중간 시간 `mid` : (최소 시간 + 최대 시간 ) / 2
- 중간 시간을 찍어보고, 그 시간동안 처리할 수 있는 사람 수를 구한다
- 총 처리 가능 인원이 n명보다 크거나 같다면 최대 시간을 줄여보고, 아니라면 최소 시간을 늘린다 -> 이 과정을 반복하여 정답을 찾아감

### 정답 코드
```
class Solution {
    public long solution(int n, int[] times) {
        long answer = 0;

        int maxTime = 0;
        for (int t : times) {
            if (t > maxTime) maxTime = t;
        } //for - maxTime

        long left = 1;
        long right = (long)maxTime * n;

        while (left <= right) {
            long mid = (left + right) / 2;
            long sum = 0;

            for (int t : times) {
                sum += (mid / t);
            } //for - sum

            if (sum >= n) {
                answer = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            } //if ~ else
        } //while - binary search

        return answer;
    } //solution
}
```

- 시간을 줄이거나 늘려나가는 과정이 처음에는 잘 이해되지 않았는데 몇 번 다시 읽어보며 생각해보니 이해가 되었다.
- 이진 탐색에서는 범위를 반씩 줄여 나가는 과정이 중요하다는 것을 배웠다.
