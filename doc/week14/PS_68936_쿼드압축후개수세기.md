## 프로그래머스 쿼드압축 후 개수 세기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/68936)

### 접근
- 분할정복 알고리즘을 사용

### 정답 코드
```
public class Solution {
    
    int[][] area;
    int[] answer = new int[2];
    
    public int[] solution(int[][] arr) {
        area = arr;
        
        divide(0, 0, arr.length);
        
        return answer;
    }
    
    private void divide(int x, int y, int size) {
        int firstValue = area[x][y];
        boolean isSame = true;
        
        for (int i = x; i < x + size; i++) {
            for (int j = y; j < y + size; j++) {
                if (area[i][j] != firstValue) {
                    isSame = false;
                    break;
                }
            }
            if (!isSame) break;
        } //for - 영역 돌기
        
        if (isSame) {
            if (firstValue == 0) answer[0]++;
            else answer[1]++;
            return;
        } //if - 압축
        
        int nextSize = size/2;
        divide(x, y, nextSize);
        divide(x+nextSize, y, nextSize);
        divide(x, y+nextSize, nextSize);
        divide(x+nextSize, y+nextSize, nextSize);
    } //divide
}
```

### 회고
- 분할정복 알고리즘을 학습한 이후로 관련 문제를 처음 푸는거라 코드를 바로 짜기가 쉽지 않았다.
- 재귀를 탈출해야 한다는 생각에 `if (size <= 1) return;` 코드를 추가하여 압축되지 않고 남은 1x1 단일 블록의 정답 카운트를 누락시킨 채 로직을 끝내버리는 문제가 발생하였다.
  1x1 사이즈일 때 탈출 조건 `size <= 1`을 적어주지 않아도 size가 1이 되면 배열 검사 시 무조건 `isSame = true`가 성립해 자연스럽게 재귀를 탈출한다는 흐름을 깨달았다.
- 재귀 함수를 설계할 때 억지로 기저 조건을 끼워 넣기보다는 내가 짠 논리 흐름이 마지막에 어떻게 동작하는지를 완벽하게 파악하는 것이 우선이 되어야 함을 배웠다.
