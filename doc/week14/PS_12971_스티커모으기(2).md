# 프로그래머스 스티커 모으기 (2)

[문제 링크](./https://school.programmers.co.kr/learn/courses/30/lessons/12971)

### 초기 접근과 문제점
- 현재 칸과 뒤의 두 칸 총 세 칸의 값을 비교하여 가장 큰 값을 선택 -> 선택하는 조건을 판단하기가 너무 까다롭다.

## 접근
- 동적 프로그래밍 이용
- 첫 번째 스티커를 떼는 경우 i는 3 ~ n-1 순회
- 두 번째 스티커를 때는 경우 i는 2 ~ n 순회
- 점화식: `dp[i] = Math.max(dp[i-1], dp[i-2] + sticker[i-1]);`

### 정답 코드
```
class Solution {
public int solution(int sticker[]) {
        int n = sticker.length;
        
        if (n == 1) return sticker[0];
        
        int[] dp1 = new int[n+1]; //0번지 버림
        int[] dp2 = new int[n+1]; //0번지 버림
        
        //1. 첫 번째 스티커를 떼는 경우
        dp1[1] = sticker[0];
        dp1[2] = dp1[1];
        for (int i = 3; i <= n-1; i++) {
            dp1[i] = Math.max(dp1[i-1], dp1[i-2] + sticker[i-1]);
        } //for - dp1
        dp1[n] = dp1[n-1];
        
        //2. 첫 번째 스티커를 떼지 않는 경우
        dp2[1] = 0;
        dp2[2] = sticker[1];
        for (int i = 3; i <= n; i++) {
            dp2[i] = Math.max(dp2[i-1], dp2[i-2] + sticker[i-1]);
        } //for - dp2
        
        return Math.max(dp1[n], dp2[n]);
    }
}
```

### 회고
- 처음에는 동적 프로그래밍으로 접근하려 했으나 점화식을 생각하지 못해 일반적인 시뮬레이션으로 구현하려 했다.
- 첫 번째 스티커를 떼지 않는 경우 두 번째 스티커를 무조건 뗀다는 가정이 바로 이해되진 않았지만 결국 `Math.max()` 함수를 통해 현재 스티커를 뗄지 말지 결정한다는 과정을 이해했다.
