# 프로그래머스 두 큐 합 같게 만들기

[문제 링크](https://school.programmers.co.kr/learn/courses/30/lessons/118667?language=java)

## 접근
- 각 큐의 합을 구하고, 그 둘의 총합을 구한 뒤 반으로 나눠 큐로 만들어야 하는 숫자(목표값)를 구함
- 큐1의 현재 총합을 구해 목표 값과 비교
- 합이 목표 값보다 크다면? -> 큐1을 pop하여 큐2에 넣기
- 합이 목표 값보다 작다면? -> 큐2를 pop한 값을 큐1에 넣기
- 목표 값과 완벽하게 같아질 때까지 과정 반복

### 정답 코드
```
import java.util.ArrayDeque;
import java.util.Arrays;
import java.util.Queue;

public class Solution {
    public int solution(int[] queue1, int[] queue2) {
        int answer = 0;
        
        long sum1 = Arrays.stream(queue1).sum();
        long sum2 = Arrays.stream(queue2).sum();
        long total = (sum1 + sum2);
        long half = total / 2;
        
        if (total % 2 != 0) return -1;
        
        Queue<Integer> que1 = new ArrayDeque<>();
        Queue<Integer> que2 = new ArrayDeque<>();
        for (int i = 0; i < queue1.length; i++) {
            que1.offer(queue1[i]);
            que2.offer(queue2[i]);
        } //for - insert que
        
        int limit = (que1.size() + que2.size()) * 2;
        
        while (answer <= limit) {
            if (sum1 == half) {
                return answer;
            } else if (sum1 > half) {
                int cur = que1.poll();
                sum1 -= cur;
                que2.offer(cur);
                answer++;
            } else {
                int cur = que2.poll();
                sum1 += cur;
                que1.offer(cur);
                answer++;
            } //if ~ else
        } //while
        
        return -1;
    }
}
```

### 회고
- while문의 탈출 조건을 너무 세부적이게 설정한 탓에 무한 루프를 돌게 되었다.
- 처음에는 반복문이 시작될 때마다 두 큐에서 무조건 `poll()`을 수행하여 원소가 증발하는 문제가 발생했다.
  원소를 어디로 이동시킬지 조건을 먼저 판별한 후에, 해당하는 큐에서만 원소를 꺼내 논리적 유실을 막아야 함을 깨달았다.
